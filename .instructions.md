# Auto-Check Exercise Backend - Agent Instructions

## Project Context
This is a production Node.js/Express backend service for automatically grading student exercises using OpenAI's Assistant API with Google OAuth2 authentication.

## Architecture Overview
- **Runtime**: Node.js 20+ (slim Docker container)
- **Framework**: Express.js with CORS enabled
- **Authentication**: Google OAuth2 + API Key + Email Whitelist
- **AI Integration**: OpenAI Assistant API (beta threads)
- **Deployment**: Docker-ready for cloud platforms (Cloud Run, etc.)

## Key Files
- `backend/server.js` - Main application logic
- `backend/package.json` - Dependencies and scripts
- `Dockerfile` - Container configuration
- `.env` - Environment variables (not committed)

### Authentication Flow
1. Check `x-api-key` header matches `EXTENSION_SECRET_KEY`
2. Validate Bearer token in `Authorization` header
3. Verify token with Google API (`/oauth2/v3/tokeninfo`)
4. Check email against `ALLOWED_EMAILS` whitelist
5. Attach `req.userEmail` for downstream use

### OpenAI Integration Pattern
```javascript
// 1. Create thread
const thread = await openai.beta.threads.create();

// 2. Add message
await openai.beta.threads.messages.create(thread.id, {...});

// 3. Create and poll run
const run = await openai.beta.threads.runs.create(...);
// Poll until completed/failed

// 4. Get response
const messages = await openai.beta.threads.messages.list(thread.id);

// 5. Cleanup
await openai.beta.threads.del(thread.id);
```

## Environment Variables

### Required
- `OPENAI_API_KEY` - OpenAI API key
- `OPENAI_PROJECT_ID` - OpenAI project ID
- `ASSISTANT_ID` - OpenAI Assistant ID for grading
- `CLIENT_ID` - Google OAuth2 client ID
- `GOOGLE_CLIENT_SECRET` - Google OAuth2 client secret
- `REDIRECT_URI` - OAuth2 redirect URI
- `EXTENSION_SECRET_KEY` - API key for extension authentication

### Optional
- `PORT` - Server port (default: 8080)
- `ALLOWED_EMAILS` - Comma-separated whitelist (configured in code)

## API Documentation

### POST /grade
**Headers**: 
- `Authorization: Bearer <google_token>`
- `x-api-key: <extension_secret>`

**Body**:
```json
{
  "items": [
    {
      "question": "What is 2+2?",
      "answer": "4"
    }
  ]
}
```

**Response**:
```json
{
  "success": true,
  "assistantText": "Grading feedback..."
}
```

### POST /auth/google
**Body**: `{ "code": "<oauth_code>" }`

**Response**: OAuth tokens (access_token, refresh_token, etc.)

### POST /auth/refresh
**Body**: `{ "refresh_token": "<refresh_token>" }`

**Response**: New access token and expiry

### POST /exchange-token
**Body**: Token exchange parameters (code or refresh_token)

**Response**: Exchanged tokens

## Security Considerations

1. **Never commit .env files** - All secrets in environment variables
2. **Email whitelist** - Only authorized emails can access /grade
3. **API key validation** - Extension must provide valid x-api-key
4. **Token verification** - Google tokens verified server-side
5. **CORS enabled** - Configure for production domains

## Troubleshooting

### Common Errors

**"OPENAI_API_KEY not set"**
- Check .env file or environment variables
- Verify variable name matches exactly

**"Email không có trong danh sách cấp phép"**
- Add user email to ALLOWED_EMAILS array
- Check email case sensitivity

**"Token exchange failed"**
- Verify GOOGLE_CLIENT_SECRET is correct
- Check redirect_uri matches Google Console

**"Run Failed"**
- Check ASSISTANT_ID is valid
- Review OpenAI run status and last_error
- Verify OpenAI API key has access to Assistant API
